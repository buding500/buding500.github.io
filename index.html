margin负值进行两栏或三栏自适应布局

发表于 2017-08-10 | 分类于 css

有个后台项目需要使用两栏自适应布局，用margin负值，同时自己也回顾一下：

两栏布局

原理：

  • left元素float之后，会自动脱离文档，main元素则会跑到left右边，
  • left元素margin-right负值之后，main容器的起始位置跟left容器一样了，这时left元素在main元素之的上层。
  • 给main元素margin-left值等于left的宽度值，left元素的负margin和main元素的正margin相抵消，main元素的起始位置在left元素的结束位置了，即实现了自适应。
三栏布局

原理：

  • 三者均使用float让自己脱离文档流。这个地方注意main元素写在最前面，给宽度100%;
  • left元素使用margin负值将自己放到main元素上方的最左侧位置；
  • right元素使用margin负值将自己放到main元素上方的右侧位置；
  • 这个时候的main元素实际是全屏宽度，位于left元素和right元素下方。给main元素一个子元素，margin左右值给left元素和right元素的宽度，那么内容区域就是main-content，显示在了最中间，且宽度自适应；
  • 这个地方的margin负值仅仅用来让left和right定位到main上方的位置，并没有像两栏布局一样是一个相抵消的过程。

mouse的那些事

发表于 2017-08-09 | 分类于 jquery

老是在写鼠标事件，却没认真仔细的区别好它们，今天来总结下它们的用法和区别。jquery里的mouse事件一共有7个，它们分别是：

mouseenter: 鼠标指针穿过被选元素时触发。只有在鼠标指针穿过被选元素时，才会触发mouserenter事件，大多数情况下与mouseleave一起使用。
mouseleave： 鼠标指针离开被选元素时触发。只有在鼠标指针离开被选元素时，才会触发mouseleave事件，大多数情况下与mouseenter一起使用。
mouseover： 当鼠标指针位于元素上方时触发。不论鼠标指针穿过被选元素或者其子元素时，都会触发mouseover事件，大多数情况下与mouseout一起使用。
mouseout： 当鼠标指针从元素上方移开时触发。不论鼠标指针离开被选元素或者其子元素时，都会触发mouseout事件， 大多数情况下与mouseover一起使用。
mousedown: 当鼠标指针移到到元素上方，并按下鼠标按键时触发。mousedown事件仅需要按键被按下，而不需要按键松开即可发生。
mouseup： 当鼠标指针在元素上方松开按键时发生，mouseup事件仅需要放松按键。
mousemove: 当鼠标指针在指定的元素中移动时触发，mousemove事件处理函数会被传递一个变量，即Event事件对象，其.clientX和.clientY属性代表鼠标的坐标。

描述写了这么多，还是来看代码比较清楚明了：

  • 对于mouseenter和mouseover的区别，只需要鼠标来回移动到所选元素和其子元素之间就可以看出来。mouseover事件在鼠标移动到子元素上时也会触发，而mouseenter则不会。
  • mousedown和mouseup只有在按键时才会触发。
  • mousemove只在要触发该事件的元素上方都会触发，Event对象事件用来记录鼠标的位置。

javascript中与parse有关的方法

发表于 2017-07-29 | 分类于 javascript

parse在javascript中是用来强制转换类型的，在javascript中有关parse的方法有以下几个：
parseInt()、parseFloat()、Date.parse()以及JSON.parse();一一来看一下:

parseInt()：解析一个字符串,并返回一个整数。

parseInt()会从第一个位置进行判断它是否是个有效数字；如果不是，则返回NaN。如果是有效数字，将继续向下查找直到发现非有效数字的字符为止，此时parseInt()将把该字符之前的字符串转换成数字。

parseFloat()：函数可解析一个字符串,并返回一个浮点数。

与parseInt()方法的处理方式相似，从第一个位置查看每个字符，到第一个非有效的字符为止，然后把该字符之前的字符串转换成数字。但是，对于这个方法来说，第一个出现的小数点是有效字符。如果有两个小数点，第二个小数点将被看作无效的。

Date.parse()：将时间转成时间戳。返回该日期与 1970 年 1 月 1 日午夜之间相差的毫秒数。

参数格式：月/日/年 时:分:秒

关于参数的问题可查看：http://www.cnblogs.com/yingcaiyi/p/4884903.html

JSON.parse()：用于从一个字符串中解析出json 对象。

在项目过程中，经常遇到一些表单操作，涉及到表单，自然逃不掉键盘事件。比如：在输入的时候隐藏验证错误提示，在最后一个输入框里输入回车的时候提交表单等等。因为现在项目用的是jquery，想说来整理一下这几个键盘事件。

jquery键盘事件有以下几个：

  • keydown()：当键盘按下时触发；
  • keypress()：当键盘按下时触发；
  • keyup()：当按键松开时触发，发生在当前获得焦点的元素上

  • 当我们一个字符一个字符输入时，三者表现一致；
  • 当我们一直按着某个字符键，keydown和keypress一直在发生变化，并且次数一致；
  • 这个时候输入框满了，我们删除一些字符。然后：区别出来啦！！！这个时候keydow发生了变化，而keypress没有变化，也就是backspace键keypress识别不了；
  • 猜测keypress不能识别功能键，然后按下ctrl、shift、tab、delete、CapsLock等功能键来试一试，果然keypress没有反应，而keydown数字一直在变化；
  • 然后我们来测试一下特殊字符，比如?<>{}[]’’””:;等，发现keypress和keydown表现一致；
  • 最后我们来试一下中中文，这时区别又出来了，keypress不能识别中文，不管是中文汉字还是中文字符（比如：？《》｛｝【】），而keydown只要按下键就会识别，keyup只要松开键也会有识别。总的来说：keypress没有办法识别双字节字符！
最后来总结一下：
  1. keydown和keypress在没有输入特殊字符的情况下表现一致；
  2. keyup只在松开键的时候触发；
  3. keypress不能识别功能键和双字节字符。

(pattern) | (?:pattern) | (?=pattert) | (?!pattern) | (?<=pattert) | (?<!pattern)的用法

发表于 2017-07-07 | 分类于 javascript

(pattern)：子表达式，也叫分组，匹配pattern并获取这一匹配。

上面例子中(11|22|33)就是一个子表达式，子表达式在引用过程中有两种写法：\1 和 $1。

  • \1 是在正则表达式本身中引用分组1的内容，也称回溯引用。比如我们要在一段html中匹配所有的标题，即<h1><h1>…<h6></h6>：
    正则：<(h[1-6])>.*?<\/\1>，(h[1-6])匹配到第一个h标题，后面\1再引用分组1这个匹配内容，即h1对应h1,h6对应h6，不会出现h1对h6的情况。如果想用分组2则用\2,依次类推。

  • $1 是在替换中调用分组的内容。比如我们需要把html里的邮箱都加上链接，匹配name@email.com然后用name@email.com来替换：
    正则：(\w+(.?)\w@(\w+)(.?)\w+(.?)\w);替换：$1

这个地方有个坑，就是new RegExp()的时候，采用字符串行式的方式 参数，一定要给里面的斜杠进行转义(比如\w写成\\w；/\写成///\)，否则会匹配不到。为了以防掉坑，以后还是直接用/pattern/的方式来定义好了。

(?:pattern)：非捕获组，它们只分组而不对分组进行捕获。

很疑惑捕获组和非捕获的具体区别，于是乎开始查各种资料，似乎明白了点什么：

凡是以问号开头的都是非捕获组。它们在匹配过程中，不对分组进行存值，也就是说匹配结果不能继续使用，而捕获分组则可以通过\1或者$1这样的方式来使用匹配到并捕获的分组结果。这样来讲，不捕获组进行匹配的效率要高。

比如我们要在一篇文章中查找”happy”和”happiness”两个单词，正则表达式可以写成/happy|happiness/，也可写成/happ(y|iness)/，但是去缓存子表达式(y|iness)没有意义，这个时候就可以用/happ(?:y|iness)/来进行非捕获性匹配，简洁又不占用资源。

(?=pattern)：向前查找，匹配后面是pattern的数据，但pattern不在结果中返回。pattern左为前，右为后。

(?!pattern)：与(?=pattern)相反，匹配后面不是pattern的数据，pattern不在结果中返回。

(?<=pattern)：向后查找，匹配前面是pattern的数据，pattern不在结果中返回。

(?<!pattern)：与(?<=pattern)相反，匹配前面不是pattern的数据，pattern不在结果中返回。

向后查找(?<=pattent)和(?<!pattern)是不支持javascript，所以我也没办法写例子了。

前后查找也有其它的叫法，叫做环视或者断言。具体可以参考这篇文章 深入理解正则表达式高级教程

正则表达式的基础用法

发表于 2017-07-05 | 分类于 javascript

近期目标是学会自己写正则表达式，于是乎，开始看《正则表达式必知必会》，基础的看得差不多了，觉得也应该多多少少记录一些下来。

正则表达式的用途:
  1. 查找特定的信息
  2. 查找并替换特定的信息

常用的无字符分类列表：

基本元字符：
字符 描述
. 匹配任意单个字符
| 匹配或操作符
[] 匹配字符集合中的一个字符
[^] 对字符集合求非
- 定义一个区间，如[a-z],放在字符集首位或末尾则匹配本身
\ 对它后面的字符进行转义

表示次数的元字符：

字符 描述
+ 匹配前面的子表达式1次或多次，至少匹配1次
* 匹配前面的子表达式0次或多次
？ 匹配前面的子表达式0次或1次
{n} n为非负整数，匹配n次
{n,} n为非负整数，至少匹配n次
{n,m} n为非负整数，匹配至少n次，最多m次

表示位置的元字符：

字符 描述
^ 匹配输入字符串的开始位置
$ 匹配输入字符串的结束位置
\b 匹配一个单词边界，也就是指单词和空格间的位置
\B 匹配非单词边界

空白元字符

字符 描述
[\b] 回退（并删除）一个字符(Backspace键)
\f 换页符
\n 换行符
\r 回车符
\t 制表符(tab键)
\v 垂直制表符

特殊元字符

字符 描述
\d 匹配一个数字字符。等价于 [0-9]
\D 匹配一个非数字字符。等价于 [^0-9]
\s 匹配任何空白字符，包括空格、制表符、换页符等等。等价于 [ \f\n\r\t\v]
\S 匹配任何非空白字符。等价于 [^ \f\n\r\t\v]
\w 匹配包括下划线的任何单词字符。等价于 [A-Za-z0-9_]
\W 匹配任何非单词字符。等价于 [^A-Za-z0-9_]
