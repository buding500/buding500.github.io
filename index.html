(pattern) | (?:pattern) | (?=pattert) | (?!pattern) | (?<=pattert) | (?<!pattern)的用法

发表于 2017-07-07 | 分类于 javascript

(pattern)：子表达式，也叫分组，匹配pattern并获取这一匹配。

上面例子中(11|22|33)就是一个子表达式，子表达式在引用过程中有两种写法：\1 和 $1。

  • \1 是在正则表达式本身中引用分组1的内容，也称回溯引用。比如我们要在一段html中匹配所有的标题，即<h1><h1>…<h6></h6>：
    正则：<(h[1-6])>.*?<\/\1>，(h[1-6])匹配到第一个h标题，后面\1再引用分组1这个匹配内容，即h1对应h1,h6对应h6，不会出现h1对h6的情况。如果想用分组2则用\2,依次类推。

  • $1 是在替换中调用分组的内容。比如我们需要把html里的邮箱都加上链接，匹配name@email.com然后用name@email.com来替换：
    正则：(\w+(.?)\w@(\w+)(.?)\w+(.?)\w);替换：$1

这个地方有个坑，就是new RegExp()的时候，采用字符串行式的方式 参数，一定要给里面的斜杠进行转义(比如\w写成\\w；/\写成///\)，否则会匹配不到。为了以防掉坑，以后还是直接用/pattern/的方式来定义好了。

(?:pattern)：非捕获组，它们只分组而不对分组进行捕获。

很疑惑捕获组和非捕获的具体区别，于是乎开始查各种资料，似乎明白了点什么：

凡是以问号开头的都是非捕获组。它们在匹配过程中，不对分组进行存值，也就是说匹配结果不能继续使用，而捕获分组则可以通过\1或者$1这样的方式来使用匹配到并捕获的分组结果。这样来讲，不捕获组进行匹配的效率要高。

比如我们要在一篇文章中查找”happy”和”happiness”两个单词，正则表达式可以写成/happy|happiness/，也可写成/happ(y|iness)/，但是去缓存子表达式(y|iness)没有意义，这个时候就可以用/happ(?:y|iness)/来进行非捕获性匹配，简洁又不占用资源。

(?=pattern)：向前查找，匹配后面是pattern的数据，但pattern不在结果中返回。pattern左为前，右为后。

(?!pattern)：与(?=pattern)相反，匹配后面不是pattern的数据，pattern不在结果中返回。

(?<=pattern)：向后查找，匹配前面是pattern的数据，pattern不在结果中返回。

(?<!pattern)：与(?<=pattern)相反，匹配前面不是pattern的数据，pattern不在结果中返回。

向后查找(?<=pattent)和(?<!pattern)是不支持javascript，所以我也没办法写例子了。

前后查找也有其它的叫法，叫做环视或者断言。具体可以参考这篇文章 深入理解正则表达式高级教程

正则表达式的基础用法

发表于 2017-07-05 | 分类于 javascript

近期目标是学会自己写正则表达式，于是乎，开始看《正则表达式必知必会》，基础的看得差不多了，觉得也应该多多少少记录一些下来。

正则表达式的用途:
  1. 查找特定的信息
  2. 查找并替换特定的信息

常用的无字符分类列表：

基本元字符：
字符 描述
. 匹配任意单个字符
| 匹配或操作符
[] 匹配字符集合中的一个字符
[^] 对字符集合求非
- 定义一个区间，如[a-z],放在字符集首位或末尾则匹配本身
\ 对它后面的字符进行转义

表示次数的元字符：

字符 描述
+ 匹配前面的子表达式1次或多次，至少匹配1次
* 匹配前面的子表达式0次或多次
？ 匹配前面的子表达式0次或1次
{n} n为非负整数，匹配n次
{n,} n为非负整数，至少匹配n次
{n,m} n为非负整数，匹配至少n次，最多m次

表示位置的元字符：

字符 描述
^ 匹配输入字符串的开始位置
$ 匹配输入字符串的结束位置
\b 匹配一个单词边界，也就是指单词和空格间的位置
\B 匹配非单词边界

空白元字符

字符 描述
[\b] 回退（并删除）一个字符(Backspace键)
\f 换页符
\n 换行符
\r 回车符
\t 制表符(tab键)
\v 垂直制表符

特殊元字符

字符 描述
\d 匹配一个数字字符。等价于 [0-9]
\D 匹配一个非数字字符。等价于 [^0-9]
\s 匹配任何空白字符，包括空格、制表符、换页符等等。等价于 [ \f\n\r\t\v]
\S 匹配任何非空白字符。等价于 [^ \f\n\r\t\v]
\w 匹配包括下划线的任何单词字符。等价于 [A-Za-z0-9_]
\W 匹配任何非单词字符。等价于 [^A-Za-z0-9_]

RegExp属性lastIndex和与正则相关的String方法

发表于 2017-07-04 | 分类于 javascript

趁热打铁，一并把RegExp对象的属性也回顾一遍吧：

属性 描述
global RegExp 对象是否具有标志 g
ignoreCase RegExp 对象是否具有标志 i
multiline RegExp 对象是否具有标志 m
lastIndex 一个整数，标示开始下一次匹配的字符位置
source 正则表达式的源文本

重点看一下lastIndex:

lastIndex属性声明的是上一次匹配文本之后的第一个字符的位置，而上次匹配的结果是由exec()方法和test()方法找到的，它们都以lastIndex属性所指的位置做为下次查找的起始点。这样的话，就可以通过反复调用这两个方法来遍历一个字符串中所有的匹配文本。
lastIndex属性是可读可写的，只要目标字符串的下一次搜索开始中，就可以对它进行设置。当exec()或者test()方法再找也不到可以匹配的文本时，它们会自动把lastIndex属性的值重置为0。

特别重要：不具有标志 g 和不表示全局模式的 RegExp 对象不能使用 lastIndex 属性。
提示：如果在成功地匹配了某个字符串之后就开始检索另一个新的字符串，需要手动地把这个属性设置为 0。

之前使用过Stirng的match()方法结合正则表达式来匹配字符串，再来看看还有哪些与正则相关的String对象方法呢？

search():用于检索字符串中指定的子字符串，或检索与正则表达式相匹配的子字符串。返回字符串中第一个匹配的位置。search() 方法不执行全局匹配，它将忽略标志 g，它同时忽略 regexp 的 lastIndex 属性。
replace():用于在字符串中用一些字符替换另一些字符，或替换一个与正则表达式匹配的子串。

replace()方法有两个参数：
regexp/substr：必需。规定子字符串或要替换的模式的 RegExp 对象。
replacement：必需。一个字符串值。规定了替换文本或生成替换文本的函数。
返回一个新的字符串，是用 replacement 替换了 regexp 的第一次匹配或所有匹配之后得到的。

如果没有指定全局标志 g ，则只替换第一个匹配子串:

replace()参数replacement 可以是字符串，也可以是函数。

split()：把一个字符串分割成字符串数组。两个参数：第一个参数是字符串或者正则表达式，用来指定从哪里开始分隔；第二个参数为可选参数，用来指定数组返回的最大长度。

正则表达式中test,exec,match方法的区别

发表于 2017-07-04 | 分类于 javascript
test(): 返回值是boolean值。查找对应的字符串中是否存在模式。
exec(): 查找并以数组形式返回匹配到的结果。如果没有匹配到模式，则返回空数组；否则返回数组只有一项的数组。

如果我们相要把所有的b3+任意字符(如b3c,b3a,b3d)匹配出来应该怎么来做呢，加上全局标志 g 来试一试：

确实匹配到了b3+任意字符的字符串，但exec()每次返回的数组只有一项，这很不方便，下面介绍match()方法，它可以解决这个问题！

match(): match()跟exec()很像，不过match()是String对象的方法，而exec()方法是RegExp对象的方法。match()参数是正则表达式，而exec()参数是要匹配的字符串，两者相反。在没有全局标志下，两者返回值相同。

不同的地方是在全局标志 g 下两者的返回值不同，exec()是只返回一项，而match会返回所有匹配到的结果组成的数组。

这样是不是非常方便了，想要匹配多个并得到结果值，用match()就可以了。

最后来做个总结：

1、test()和exec()是RegExp对象方法，而match()是String对象方法；
2、test()和exec()参数是要匹配的字符串，而match()参数是正则表达式；
3、test()返回值为true或者false，exec()和match()返回一个数组；
4、exec()和match()在非全局标志g下均返回长度为1的数组；而在全局标志g下，exec()返回匹配结果的第一项，而match()返回匹配结果的所有项。

数组操作方法

发表于 2017-07-02 | 分类于 javascript
concat()：基于当前数组中的所有项创建一个新的数组。该方法会先创建当前数组的一个副本，然后把接收到的参数添加到这个副本的末尾，返回新构建的粗数组，并且不影响原有的数组。
slice()：基于当前数组中的一个或者多个项创建一个新的数组。该方法接收一个或者两个参数，即要返回项的起始和结束位置。一个参数时，返回从该参数指定位置开始到当前数组末尾的所有项；两个参数时，返回起始位置和结束位置之前的项，但是不包括结束位置的项。slice()方法也不影响原始数组。
splice()：返回一个从原始数组中删除的项的数组，会改变原始数组。如果没有删除项，则返回一个空数组。它根据参数的不同有以下三种用法：
删除： 可以删除任意数量的项，需要两个参数：要删除的第一项的位置,删除的项数。
插入： 可以向指定的位置插入任意数量的项，需要三个参数：起始位置,0,要插入的项。
替换： 可以向指定位置插入任意数量的项，并且同时删除任意数量的项，需要三个参数：起始位置,要删除的项数,要插入的任意数量的项。要插入的项数不必与要删除的项数相等。

es5中还增加一些迭代方法，因为平时工作中没怎么用过，在这里也只做个了解：

every()：对数组中的每一项运行给定函数，如果该函数对每一项都返回true，则返回true。

filter()：对数组中的每一项运行给定函数，返回该函数会返回true的项组成的数组。

forEach()：对数组中的每一项运行给定函数，没有返回值。

map()：对数组中的每一项运行给定函数，返回每次函数调用的结果组成的数组。

some()：对数组中的每一项运行给定函数，如果该函数对任一项返回true，则返回true。

数组项操作方法

发表于 2017-07-01 | 分类于 javascript
push()：向数组末尾添加任意项，返回数组长度。
pop()：从数组的末尾移除一项，返回移除的项。
unshift()：在数组头部添加任意项，返回数组长度。
shift()：移除数组第一项，返回移除的项。

有了这四个方法，我们就可以在数组的头和尾进行一些操作，比如模拟队列，在尾部添加项，在头部移除项，这时候只用使用push()和shift()就可以了。

上面的反向操作也是可以实现的，即在数组的头部添加项，在尾部移除项，使用unshift()和pop();

reverse()：反转数组项的顺序，返回排序后的数组。
sort()：按升序排列数组项，返回排序后的数组。

看到这个结果是不是很诧异？sort()方法会调用数组每项的toString()方法，然后将得到的字符串进行比较来进行排序。当然这不是我们想要的结果，怎么办呢? sort()方法可以接收一个比较函数作为参数，让我们来指定如何排序。

indexOf() 和 lastIndexOf()：两者均接收两个参数：要查找的项和查找起始位置(可选)；两者都返回查找项在数组中的位置，在没有找到的情况下返回-1。indexOf()是从数组的开头向后查找，lastIndexOf()则是从数组的末尾向前查找。
