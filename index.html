vue bulid指定文件路径 + 去掉生成的路径中的#

发表于 2017-10-28 | 分类于 vue
vue build修改生成文件目录名

之前把做的vue项目build之后，把dist文件夹下的东西扔给后端，后端想要放到他们特有的文件夹下而不是从根目录开始，然后路径就对应不上了。要把build的时候的路径改一下，改成waps下。

打开根目录下config文件夹下的index.js，找到build对象里的assetsPublichPath:’/‘，将这里的/改成我们需要的文件夹名字assetsPublichPath:’/waps/‘， 再重新build，生成的index里的路径就变成/waps/的了。这样就解决了前后端路径不统一的问题了。

去掉vue生成路径中的

vue项目访问的时候路径里有个#，这种情况是因为在入口js文件中，如果你不更改设置的话，vue会默认使用hash模式，该模式下回将路径格式化为 # 开头。怎么去掉呢？

在路由配置里加上： mode: ‘history’,

这是官方文档介绍：[https://router.vuejs.org/zh-cn/essentials/history-mode.html]
